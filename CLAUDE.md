# CLAUDE.md — Shovel-head-Atlas 恒久指示

このリポジトリで作業する際、Claude Code が常に守るべきルール。

## データ品質ルール(絶対原則)

### 出典URLのない事実は書かない
- `failure_modes` と `history_ja` の主張には必ず `sources` を1件以上つける
- 「一般常識」だと思っても書かない。出典が探せないなら書かない
- 「情報が薄い状態」はOK。「出典なしの推測」はNG

### 転載禁止
- マニュアルやカタログの図・本文は転載しない
- 図は自作SVGのみ (`assets/svg/`)
- 実写トレースは自分で撮った写真からのみ可
- ハーレー社の商標(ロゴ等)は使わない。名称は記述的使用に留める
- 著作権物(マニュアルPDF、カタログ画像)のバイナリをリポジトリに入れない

## データ形式ルール

- **`id` は kebab-case** (小文字英数 + ハイフン、`^[a-z0-9]+(-[a-z0-9]+)*$`)
- **`accessed` は取得日 YYYY-MM-DD**
- **OEMパーツナンバー** は大文字英数 + ハイフンのみ (`26204-73` 形式)
- **年式範囲** は `[開始, 終了]` の2要素配列 (1966..1984)
- **要約は日本語**、ソースリンクは原語のまま

## 情報の信頼性表現

- 情報の正誤に自信がない場合は `"confidence": "low"` を付けて残す(消さない)
- 「報告が多い/散見される」レベルの表現 + `severity` + 出典数で信頼性を示す
- 頻度の数値化はしない(フォーラム発の伝聞が多いため)

## 開発姿勢

- 素の HTML + CSS + JS 縛り(ビルド不要、フレームワーク禁止)
- 依存パッケージを増やすときは事前に相談
- SPEC.md に無い決定を独断で行わない
- Phase の順序 (0→1→2→3→4) を守り、飛び越し実装しない
